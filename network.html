IP (Internet Protocol)

Full Form: Internet Protocol
Function: Provides logical addressing and routing for data packets across interconnected networks.

IP defines how data packets should be addressed, transmitted, and delivered across networks. Every device connected to a network is assigned a unique IP address that identifies it logically, ensuring that packets reach the correct destination.

  • Addressing: IP assigns each device a unique numerical label (e.g., 192.168.1.1) for identification.
  • Routing: Routers use IP addresses to determine the most efficient path for data transmission.
  • Connectionless: IP doesn‚Äôt maintain a continuous connection ‚Äî it sends packets independently.
Example: When you load a webpage, IP ensures that the data from the server (e.g., 142.250.191.174 for Google) reaches your computer through multiple routers that forward the packets using the best route.

IPv4 vs. IPv6

Function: These are versions of the Internet Protocol that define how devices are addressed on a network.

  • IPv4: Uses 32-bit addresses (e.g., 192.168.1.1), supporting about 4.3 billion unique addresses. It‚Äôs still widely used but limited in capacity due to global device growth.
  • IPv6: Uses 128-bit addresses (e.g., 2001:0db8:85a3::8a2e:0370:7334), allowing for trillions of unique identifiers. It also improves efficiency and security with simpler header structures and built-in encryption.
Example: Modern ISPs and mobile networks increasingly rely on IPv6 to accommodate billions of new IoT devices, ensuring each one can have a globally unique address.

DHCP (Dynamic Host Configuration Protocol)

Full Form: Dynamic Host Configuration Protocol
Function: Automatically assigns IP addresses and network configuration parameters to devices.

DHCP simplifies network management by dynamically distributing network settings to devices when they connect. Instead of manually configuring IPs, subnet masks, and gateways, the DHCP server does it all automatically.

  • Automation: Devices (clients) request configuration when they join the network.
  • DORA Process: Discover ‚Üí Offer ‚Üí Request ‚Üí Acknowledge.
  • Lease Time: IP addresses are temporary and periodically renewed.
Example: When you connect your laptop to Wi-Fi, it sends a DHCP Discover broadcast. The router‚Äôs DHCP server offers an IP (e.g., 192.168.0.101), your laptop requests it, and the router acknowledges ‚Äî instantly configuring your device.

DNS (Domain Name System)

Full Form: Domain Name System
Function: Translates human-readable domain names into machine-readable IP addresses.

The DNS is a distributed database that maps domain names to IP addresses. It allows users to type names like www.example.com instead of remembering complex IP numbers.

  • Resolution Process: When you enter a URL, your device queries DNS servers in order ‚Äî local cache, recursive resolver, root server, TLD server, and authoritative server ‚Äî until it finds the correct IP.
  • Hierarchy: Organized into layers (.com, .org, .net) and managed globally.
  • Caching: DNS results are cached locally for faster future lookups.
Example: When you type www.google.com into your browser, your computer asks a DNS resolver for its IP. The resolver returns something like 142.250.191.174, which your browser then uses to connect to Google‚Äôs web server.
