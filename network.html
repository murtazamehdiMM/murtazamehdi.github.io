IP (Internet Protocol)

Full Form: Internet Protocol
Function: Provides logical addressing and routing for data packets across interconnected networks.

IP defines how data packets should be addressed, transmitted, and delivered across networks. Every device connected to a network is assigned a unique IP address that identifies it logically, ensuring that packets reach the correct destination.

  • Addressing: IP assigns each device a unique numerical label (e.g., 192.168.1.1) for identification.
  • Routing: Routers use IP addresses to determine the most efficient path for data transmission.
  • Connectionless: IP doesn‚Äôt maintain a continuous connection ‚Äî it sends packets independently.
Example: When you load a webpage, IP ensures that the data from the server (e.g., 142.250.191.174 for Google) reaches your computer through multiple routers that forward the packets using the best route.

IPv4 vs. IPv6

Function: These are versions of the Internet Protocol that define how devices are addressed on a network.

  • IPv4: Uses 32-bit addresses (e.g., 192.168.1.1), supporting about 4.3 billion unique addresses. It‚Äôs still widely used but limited in capacity due to global device growth.
  • IPv6: Uses 128-bit addresses (e.g., 2001:0db8:85a3::8a2e:0370:7334), allowing for trillions of unique identifiers. It also improves efficiency and security with simpler header structures and built-in encryption.
Example: Modern ISPs and mobile networks increasingly rely on IPv6 to accommodate billions of new IoT devices, ensuring each one can have a globally unique address.

DHCP (Dynamic Host Configuration Protocol)

Full Form: Dynamic Host Configuration Protocol
Function: Automatically assigns IP addresses and network configuration parameters to devices.

DHCP simplifies network management by dynamically distributing network settings to devices when they connect. Instead of manually configuring IPs, subnet masks, and gateways, the DHCP server does it all automatically.

  • Automation: Devices (clients) request configuration when they join the network.
  • DORA Process: Discover ‚Üí Offer ‚Üí Request ‚Üí Acknowledge.
  • Lease Time: IP addresses are temporary and periodically renewed.
Example: When you connect your laptop to Wi-Fi, it sends a DHCP Discover broadcast. The router‚Äôs DHCP server offers an IP (e.g., 192.168.0.101), your laptop requests it, and the router acknowledges ‚Äî instantly configuring your device.

DNS (Domain Name System)

Full Form: Domain Name System
Function: Translates human-readable domain names into machine-readable IP addresses.

The DNS is a distributed database that maps domain names to IP addresses. It allows users to type names like www.example.com instead of remembering complex IP numbers.

  • Resolution Process: When you enter a URL, your device queries DNS servers in order ‚Äî local cache, recursive resolver, root server, TLD server, and authoritative server ‚Äî until it finds the correct IP.
  • Hierarchy: Organized into layers (.com, .org, .net) and managed globally.
  • Caching: DNS results are cached locally for faster future lookups.
Example: When you type www.google.com into your browser, your computer asks a DNS resolver for its IP. The resolver returns something like 142.250.191.174, which your browser then uses to connect to Google‚Äôs web server.

What is Cisco Packet Tracer?

Cisco Packet Tracer is a sophisticated network simulation and visualization tool developed by Cisco Systems. It enables users to design, configure, and troubleshoot complex network topologies virtually ‚Äî without requiring any physical networking equipment.

The tool replicates real-world networking environments by allowing users to add devices such as routers, switches, computers, and servers. It also supports packet-level simulations, so you can observe how data travels across networks and how various protocols interact.

Packet Tracer is widely used in Cisco‚Äôs Networking Academy and by network professionals to build their conceptual understanding, test configurations safely, and prepare for certifications like CCNA (Cisco Certified Network Associate).

What is the Ping Command?

The ping (Packet Internet Groper) command is a diagnostic network tool used to test connectivity between two devices on a network. It works by sending ICMP Echo Request packets to a target device and waiting for ICMP Echo Reply responses.

The ping command helps determine if a destination host is reachable, how long it takes for data to travel to and from that host (latency), and whether packet loss is occurring. It‚Äôs one of the simplest yet most powerful tools for verifying basic network communication and troubleshooting connection issues.

Example: If PC A sends a ping to PC B using the address 192.168.1.2, and PC B replies successfully, it confirms that both computers are correctly configured and communicating on the same network.

We begin by opening Cisco Packet Tracer and placing two end devices ‚Äî PC A and PC B ‚Äî onto the workspace. These will represent two computers on the same local network.

Phase 2: Connecting the Devices

Phase 3: Assigning IP Addresses

Each computer is assigned a unique IPv4 address within the same network.

  • PC A ‚Üí 192.168.1.1

Both IPs are part of the same subnet (255.255.255.0 by default), which means they can communicate directly without any routing device. Once assigned, the IP configuration should be verified using the desktop settings inside Packet Tracer.

This is the fundamental concept of IP connectivity ‚Äî devices communicate using IP addresses, and the ping command validates that communication path.

In this second practical, we will explore how the Dynamic Host Configuration Protocol (DHCP) works in a simulated network using Cisco Packet Tracer. The goal is to understand how a DHCP server automatically assigns IP addresses and other network information to client devices on a local network.

What is a Server?

A server is a specialized computer that provides services or resources to other devices on a network, known as clients. In this practical, the server distributes IP addresses automatically using DHCP. It acts as a central point for managing configurations, improving efficiency and reducing human error in large networks.

What is a Switch?

A network switch is a device that connects multiple computers, servers, or other networked devices within the same local area network (LAN). It operates at the data link layer (Layer 2) of the OSI model and forwards data based on MAC addresses. In our setup, the switch ensures all connected devices can communicate efficiently with each other.

Gateway Address

The default gateway is the IP address of the device that routes traffic from a local network to other networks or the internet. It acts as the exit point for data leaving the LAN. In this simulation, our gateway address will represent the network‚Äôs virtual router.

Address Pool / Range

An address pool (or IP range) defines the set of IP addresses that a DHCP server can distribute to clients automatically. Once configured, the server assigns available addresses from this range to devices that request network access.

Phase 1: Preparing the Devices

We began by setting up the network in Cisco Packet Tracer. Three computers‚ÄîPC A, PC B, and PC C‚Äîwere placed alongside one server labeled Server 0. All devices were connected to a central switch using straight-through Ethernet cables.

Phase 2: Configuring the Server

Next, we opened Server 0 and assigned it static network settings. These included an IP address, a subnet mask, a gateway address for routing, and a DNS address to resolve domain names. The gateway and DNS values are crucial for directing traffic and translating hostnames into IP addresses.

Phase 3: Enabling and Configuring DHCP Service

In this phase, we navigated to the Services tab of the server and enabled the DHCP service. The configuration was as follows:

Once activated, the server became responsible for automatically distributing IP addresses to all devices that requested them within this network.

Phase 4: Assigning IPs and Testing Connectivity

Each client device (PC A, PC B, and PC C) was configured to obtain an IP address automatically through DHCP. Unlike the manual configuration in our previous experiment, each computer now received its IP, gateway, and DNS information directly from the server.

To verify the configuration, we used the ping command from PC A to test connectivity to the server. The successful reply confirmed that DHCP had correctly assigned IP addresses and that network communication was functioning as expected.
