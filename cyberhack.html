Offensive Cyber: Password Cracking Simulation

Ethical demonstration of password-protected ZIP cracking using Kali Linux.

⚠️ Ethical & Legal Disclaimer

All techniques demonstrated here were performed in a controlled, isolated virtual environment using Kali Linux. Unauthorized access, password cracking, or any form of cyber intrusion on systems you do not own or have explicit permission to test is illegal in most jurisdictions and unethical. This tutorial is for educational and defensive security purposes only. Always obtain written consent before testing any system.

Part 1: Preparing the Target File

For this simulation, we created a simple text file named findme.txt inside the directory:
~/Desktop/Linux/Learn_C/

This path is not mandatory but keeps our practice organized and isolated from the rest of the system.

user@kali:~/Desktop/Linux/Learn_C$ echo "Secret message: Welcome to the vault!" > findme.txt
user@kali:~/Desktop/Linux/Learn_C$ ls

We created findme.txt and verified its existence using ls.

Part 2: Password-Protecting the File

We now compress and encrypt findme.txt into a password-protected ZIP archive using the zip command with the -e flag (encrypt).
Password used: admin1234 (simple for demonstration).

user@kali:~/Desktop/Linux/Learn_C$ zip -e vault.zip findme.txt
user@kali:~/Desktop/Linux/Learn_C$ rm findme.txt
user@kali:~/Desktop/Linux/Learn_C$ ls

The original file is deleted to simulate a "locked vault" scenario. Only vault.zip remains.

Part 3: Extracting the Password Hash

zip2john is a tool from the John the Ripper suite that extracts the encrypted hash from a ZIP file.
A hash is a one-way mathematical representation of the password — we cannot reverse it, but we can guess until we match it.

user@kali:~/Desktop/Linux/Learn_C$ zip2john vault.zip > password.hash
user@kali:~/Desktop/Linux/Learn_C$ ls

Output: vault.zip and password.hash — ready for cracking.

Placeholder for Screenshot 3 (917x567) - Hash extraction

Part 4: Wordlist Attack with John the Ripper

John the Ripper is a password cracking tool. We use a wordlist — a list of common passwords — to guess the ZIP password.
rockyou.txt is a famous leaked wordlist from a 2009 breach containing over 14 million real passwords.

You can also create custom wordlists using tools like crunch or CeWL.

user@kali:~/Desktop/Linux/Learn_C$ john --wordlist=/usr/share/wordlists/rockyou.txt password.hash

Result: Password cracked: admin1234

Part 5: Unlocking the Vault

With the password recovered, we can now extract the original file using unzip.

user@kali:~/Desktop/Linux/Learn_C$ unzip vault.zip
password: admin1234
user@kali:~/Desktop/Linux/Learn_C$ ls
user@kali:~/Desktop/Linux/Learn_C$ cat findme.txt

Success! The secret message is revealed: "Secret message: Welcome to the vault!"

Conclusion

This simulation demonstrated a full password cracking workflow: from encryption to hash extraction, wordlist attack, and successful decryption.

Remember: Only use these skills defensively. Strengthen your own systems. Teach others. Never harm.

Ethical hacking is about protection, not destruction. Stay on the right side of the law — and the keyboard.
