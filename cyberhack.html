John, the Cyber Ripper

Cracking password-protected ZIP files using John the Ripper in Kali Linux.

Warning: All the tests and simulations on this page were performed in controlled and ethical environments. Performing such acts on unauthorized systems is extremely unprofessional and, in some cases, may be illegal. This tutorial is for educational purposes only — to understand how weak passwords can be exploited and why strong security matters.

What is John the Ripper?

John the Ripper — often just called john — is one of the most powerful and widely respected password-cracking tools in cybersecurity. Originally developed in 1996, it’s now a staple in Kali Linux and used by ethical hackers, penetration testers, and security researchers worldwide.

Think of it as a digital lockpicker. It doesn’t “guess” passwords randomly — instead, it uses smart techniques like:

  • Dictionary attacks: Trying common words and phrases from a wordlist (like rockyou.txt, a famous leaked password database).
  • Brute-force: Systematically trying every possible combination (slow but thorough).
  • Hybrid attacks: Combining wordlists with rules (e.g., adding “123” to the end of words).

In this tutorial, we’ll use John to crack a password-protected ZIP file — a common real-world scenario in penetration testing when recovering or auditing archived data.

Phase 1: Setting Up Our Lab Environment

To keep our learning organized and separate from the rest of the system, we created a dedicated workspace:

  • Navigated to the Desktop directory.
  • Created a folder called Linux.
  • Inside it, made a subfolder named Learn_C.

Inside Learn_C, we created a simple text file called findme.txt and added some secret content. This file will be our "treasure" — but soon, it’ll be locked away.

Phase 2: Creating a Password-Protected ZIP Archive

Using the built-in zip tool, we encrypted our findme.txt file into a password-protected archive:

zip -e vault.zip findme.txt

The -e flag tells zip to encrypt the file and prompt for a password. We chose a simple one on purpose: admin1234 — so the cracking process finishes quickly for demo purposes.

Now, vault.zip contains our encrypted text file. Without the password, the contents are inaccessible.

XXXX — Screenshot of zip command and password prompt

Phase 3: Cleaning Up & Verifying the Archive

To avoid confusion and simulate a real recovery scenario, we removed the original unencrypted file:

rm findme.txt

Then, we listed the directory contents:

ls

Result: Only vault.zip remains. The original file is gone — now the only way to access the data is by cracking the ZIP password.

XXXX — Screenshot showing only vault.zip after cleanup

Phase 4: Extracting the Password Hash

John the Ripper cannot crack ZIP files directly — it needs the password hash first. A hash is a one-way mathematical fingerprint of the password.

We used zip2john — a helper tool that comes with John — to extract the hash from vault.zip:

zip2john vault.zip > password.hash

This command reads the encryption metadata from the ZIP and saves the hash into a file called password.hash.

After running ls, we now see two files: vault.zip and password.hash.

XXXX — Screenshot of zip2john output and ls showing both files

Phase 5: Cracking the Password with John

Time to unleash John the Ripper. We launched a dictionary attack using the famous rockyou.txt wordlist — a collection of over 14 million real leaked passwords.

john --wordlist=/usr/share/wordlists/rockyou.txt password.hash

John began testing passwords at high speed. Within seconds, it successfully cracked the password: admin1234.

Pro Tip: In real engagements, attackers often build custom wordlists using social engineering — gathering info about the target (birthdays, pet names, favorite teams) to create highly effective password guesses.

XXXX — Screenshot of John cracking the password

Phase 6: Accessing the Protected File

With the password in hand, we can now unlock the archive:

unzip vault.zip

The system prompted for the password. We entered admin1234, and the file extracted successfully — findme.txt is back!

XXXX — Screenshot of unzip prompt and successful extraction

Phase 7: Revealing the Hidden Message

Finally, we displayed the contents of the recovered file:

cat findme.txt

The secret message inside reads: "Congratulations! You've just performed a full password recovery chain."

This demonstrates how easily weak passwords fall — even in encrypted archives.

Conclusion

You've just walked through a complete ethical password recovery workflow using John the Ripper. From hash extraction to successful cracking — every step mirrors real-world penetration testing techniques.

Key Takeaway: Weak passwords are the #1 security vulnerability. Even encrypted files are only as strong as their password.

Final Warning: Never use these techniques on systems or files you do not own or have explicit permission to test. Unauthorized access is a serious crime. Always operate within legal and ethical boundaries.
