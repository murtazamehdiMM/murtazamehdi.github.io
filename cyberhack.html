Offensive Cyber: Password Cracking Simulation

Ethical demonstration of password-protected ZIP cracking using Kali Linux.

Ethical & Legal Disclaimer

All techniques demonstrated here were performed in a controlled, isolated virtual environment using Kali Linux. Unauthorized access, password cracking, or any form of cyber intrusion on systems you do not own or have explicit permission to test is illegal in most jurisdictions and unethical. This tutorial is for educational and defensive security purposes only. Always obtain written consent before testing any system.

Part 1: Preparing the Target File

We begin by creating a plain text file named findme.txt in the directory:
~/Desktop/Linux/Learn_C/

This path is optional but recommended to keep our practice organized and isolated from the rest of the system.

Command used:
echo "Secret message: Welcome to the vault!" > findme.txt
Followed by ls to confirm the file was created.

[ Screenshot 1: Creating and Verifying the File ]

Part 2: Password-Protecting the File

We now compress and encrypt findme.txt into a password-protected ZIP archive.

Tool: zip — a command-line utility for creating compressed archives.
Command: zip -e vault.zip findme.txt
The -e flag enables encryption and prompts for a password.

Password used: admin1234 (intentionally weak for educational purposes).

After creation, we remove the original file:
rm findme.txt
Then use ls to confirm only vault.zip remains.

[ Screenshot 2: Creating and Securing the ZIP Archive ]

Part 3: Extracting the Password Hash

To crack the password, we first need to extract its hash.

Term: Hash — A one-way mathematical transformation of the original password. It cannot be reversed, but if we guess the correct password, we can generate the same hash and confirm a match.

Tool: zip2john — A utility from the John the Ripper suite that extracts password hashes from ZIP files.
Command: zip2john vault.zip > password.hash

This saves the hash into a file named password.hash.
We then run ls to confirm both files exist.

[ Screenshot 3: Extracting the Password Hash ]

Part 4: Wordlist Attack with John the Ripper

We now attempt to recover the password using a wordlist attack.

Term: Wordlist — A text file containing thousands or millions of common passwords, leaked credentials, or dictionary words used to guess passwords systematically.

Tool: John the Ripper — One of the most widely used open-source password cracking tools. It supports multiple attack modes, including wordlist-based guessing.

Wordlist used: rockyou.txt
A famous wordlist from a 2009 data breach containing over 14 million real passwords. It is included by default in Kali Linux at:
/usr/share/wordlists/rockyou.txt

Command:
john --wordlist=/usr/share/wordlists/rockyou.txt password.hash

Result: The password admin1234 was successfully discovered.

[ Screenshot 4: Password Cracked Using John ]

Part 5: Unlocking the Vault

With the password now known, we can extract the contents of the ZIP archive.

Command: unzip vault.zip
When prompted, enter: admin1234

After extraction, we run:
ls — to confirm findme.txt is back
cat findme.txt — to display the secret message

Success! The message reads:
"Secret message: Welcome to the vault!"

Conclusion

This simulation demonstrated a complete offensive cybersecurity workflow: from creating a protected file to cracking its password using real-world tools and techniques.

Remember: These skills are for defense only. Use them to test and secure your own systems — never to harm others.

Ethical hacking is about protection, not exploitation. Stay legal. Stay ethical. Stay sharp.
